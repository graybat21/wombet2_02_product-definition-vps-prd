# WOMBET2 제품 요구사항 문서 (MVP PRD) v0.2
- Owner 팀: Wombet2 Product Team
- 최종 업데이트: 2026-04-22

## 1. 개요·목표 (Goals & Metrics)
- **문제 정의(Pain)**: 
  - (B2C) 기존 미국 E-commerce의 범용적 5점 별점 및 정보 긍정 편향으로 인해, 부상 위험군 타겟이 치명적인 장비 결함을 찾기 위해 평균 2시간의 탐색 피로도를 감당해야 함.
  - (B2B) 길이 위주의 단순 피팅으로 인해 발볼 및 어퍼 장력 충돌 예측 실패 → NRF 기준 반품 역물류 비용 매몰 (통상 주문가의 21% 규모).
- **목표(Desired Outcome)**: 
  - (B2C) 철저하게 네거티브(단점/상성 불일치) 경고에 집중한 필터 설계로 유저에게 극적 안도감을 부여 → 최단시간 제휴 결제 전환 창출.
  - (B2B) 히트맵 기반 핏 사전 진단 모델(Wizard of Oz)로 매장의 신발 반품 건수 및 물류 낭비 자본을 방어.
- **성공 지표(북극성/보조 KPI)**:
  - **[북극성 KPI] 직구 제휴(Affiliate) 결제 전환율(CVR)**
    - 기준선(Baseline): 기존 일반 커머스 전환율 수준 (1~2% 내외)
    - 목표(Target): 단점 노출 후 결제 유도 인원 대비 **최저 5.0% 이상 달성 방어**
    - 측정 창구: Amazon Associates Dashboard, 제품 Outlink Click 추적기(Mixpanel)
  - **[보조 KPI] B2B 시범 매장 반품 감축률**
    - 기준선(Baseline): 글로벌 신발 반품률 약 24% (True Fit 조사치 활용)
    - 목표(Target): 위젯 도입 후 매장 단위 월 실측 반품 신청 건수 **20% 하향 감축**
    - 측정 창구: 제휴 매장 위젯 성과 및 관리자 Excel 대시보드 취합

## 2. 사용자와 페르소나
- **C1 김러닝 (핵심 지표 견인 / 가스라이팅 회피자)** : 인플루언서 맹신 구매 후 부상 등 극심한 고통 통과. 명확하고 뼈아픈 단점 필터가 있으면 즉시 신뢰하여 구매.
- **E1 윤양발 (B2B 지불 근거 / 반품 킬러)** : 통상적 스펙으로는 통증을 못 피해 10결레 중 8결레를 환불함. 구매 여부 전 핏/발등 압박 체크 히트맵이 절실.
- **배제 결정(Out of Scope) – N1 조미라** : 부정적 정보 및 리뷰 팩트를 꺼리고 오직 유행/디자인 위주 결정. 제품의 결함 노출 시 불쾌감 이탈 특성이 강하여 본 PRD 마케팅 타겟에서 '고의 배제'.

## 3. 사용자 스토리와 수용 기준(AC, Acceptance Criteria)
- **Story 1 (네거티브 경고 필터 노출)**
  - *Story*: As a C1 유저, I want 제품 상세 페이지 최상단에서 내 뼈를 상하게 할 결정적 단점 3가지를 확인하기를, so that 지루한 외부 리뷰 교차검증 없이 즉각 안도를 얻고 결제를 확정(Amazon Affiliate 연결) 할 수 있다.
  - *AC 1.1 (성공-렌더링 SLO)*: Given 기기 및 네트워크 정상 접속 시, When 제품 상세 페이지 렌더링 요청, Then **500ms 이내**로 붉은색 결함 경고 컴포넌트(3개 목록)가 뷰포트 최상단 영역에 100% 렌더링되어야 함.
  - *AC 1.2 (실패-Null Data 방어)*: Given 특정 기어의 DB가 비어있는 예외 건 접근 시, When 상세화면 렌더링 시, Then 빈 화면/에러 대신 회색 톤의 "[데이터 파싱/분석 중] 안전 리뷰 대기열 등록됨" 문구를 띄워 이탈을 통제해야 함.
  - *AC 1.3 (실패-제휴 만료 Fallback)*: Given Amazon 타겟 URL이 재고소진/404코드 발생 시, When 결제(Buy) 라우팅 버튼 클릭 시, Then 사용자에게 브로큰 링크를 띄우지 않고, "해당 스펙 재고 소진" 모달을 띄운 뒤 즉시 **'동일 핏/용도 점수 대체군 상품 3종 추천 링크'**로 Fallback 자동 전환해야 함.

- **Story 2 (오즈의 마법사형 사전 핏 진단 폼 운영)**
  - *Story*: As a 핏 에러 피해자 E1, I want 결제 1단계에서 핏/장력 충돌부를 히트맵으로 판별받기를, so that 착화 후 몰려오는 통증 및 피로한 환불 CS 과정을 사전에 완전히 잘라낼 수 있다.
  - *AC 2.1 (성공-진단 속도 SLA)*: Given 평발 여부 등을 기입한 Typeform 진단 시트 제출 시, When 백오피스 접수 후, Then 내부 운영망 통해 1차 수동 히트맵 결과 발송을 최대 10시간 이내 완료해야 함.
  - *AC 2.2 (실패-SLA 지연 대응)*: Given 시스템 바이럴 폭발로 10시간 내 수동 처리가 불가능한 부하 시점, When SLA 마감 임계 도달, Then Slack Webhook을 통해 경고 알림 관리자 발송 및 고객 대상 "정밀 핏 매핑 분석 지연(최대 N시간 추가)" 자동 양해 이메일 송출 보장.

## 4. 기능 요구사항(Functional)
> 💡 *개발 지침: 아래 MSCW 백로그는 개발 장기화를 배제하고 코딩을 최소화한 No-Code (Carrd, Typeform, Airtable) + API 연동을 기반으로 **1 스프린트(1.5주 내) 내에 검증 가능하도록 설계**되었습니다.*
- **[Must] 네거티브 단점 3요소 정보 렌더링 (UI/DB 연동)**
  - *근거*: 안도감 필터(Blemishing Effect)를 제공해 압도적 체류단축 및 단기 전환 CVR 방어율 형성.
  - *의존성*: Amazon Affiliate 상품 API와 내부 수작업 추출 단점 DB 매핑.
- **[Must] 핏 진단폼 Inbound 위젯 접수기 (Typeform 연동)**
  - *근거*: B2B 지불형 SaaS MVP 입증을 위해, 초기 AI 모델 개발비를 0원으로 통제하고 WTP(지불 의사)만을 확인.
  - *의존성*: 인간 휴먼 리소스(1인 운영 전담자) 처리 속도 능력 및 정확성.
- **[Should] 운동 목적(러닝/크로스핏 등) 변경 시 역산 가중치 적용 기능**
  - *근거*: 타겟(A1)에게 목적에 따라 기존 평가가 뒤바뀌는 차별적 카타르시스 입증.
  - *의존성*: 1 스프린트 내에는 단 2개의 종목(러닝, 기타리프팅)만 매핑한 프로토타입.
- **[Won't] (의도적 배제) 노란색 5점 만점 시스템, 커뮤니티, 로그인 시스템 개발 무기한 연기**

## 5. 비기능 요구사항(NFR, Non-Functional Requirement)
- **성능**: p95 프런트 경고 패널 노출 < 500ms (지연 시 CVR 수치 직접 손상).
- **가용성/보안**: 
  - (가용성) 외부 제휴망 연동 버튼 클릭 라우팅 서비스 구동 99.9%. 장애 발생 시 Sentry/Datadog으로 *Critical Threshold 알림*.
  - (보안) 수동 폼 분석 시 개인 족형 콤플렉스와 이메일 분리하여 해시맵 암호화 관리(비식별 조치).
- **모니터링 체계 강화**:
  - 마케팅 비용 임계치 타격: Meta Ads CPA(회원/리드 획득 단가)가 건당 $3.0초과 시, Slack Webhook으로 퍼포먼스 담당자 캠페인 중단 알람 통보.
  - 브로큰 링크 모니터링: Amazon Affiliate Outlink 버튼의 403/404 HTTP Error Rate 반점이 1% 초과 시 즉시 Failback 동작 검사.

## 6. 리스크·가정 (ADR 및 위험 기탁 관리)
- **[ADR-001:제품 구조 설계 결정] 3D 스캔 AI 기술 배제 및 수작업형 판독 엔진 도입 확정**
  - **설계 이유**: 머신러닝 비전 엔진을 도입하면 MVP 런칭 기간이 최소 6개월 표류함. 비즈니스 첫 가설은 "AI 기술 작동 여부"가 아니라, "**안심/위험 필터를 노출 시 B2B 매장이 반품 절감 효과로 예산을 지불하는가**" 여부. 이를 실험하려면 운영자가 직접 사진/치수를 육안 검토해 빠르게 응답하는 통제 방식이 자본 통제에 100배 적절함.
- **비즈니스 주요 리스크 및 관리 방안**:
  1. *[Risk 1: 벤더사 Affiliate 승인 배제/마찰]* (심각도: High) 
     경고를 '무지성 비난(Defamation)'으로 해석 시 제휴 권한 즉시 박탈. 
     → **대응책**: 정보 가이드라인 자체를 "보행/의학 분석"에 근거한 특정 상성(예: 평발충돌)으로 팩트 박스화 하여 논란 원천소거.
  2. *[Risk 2: 운영망 수용 폭발로 CS SLA 파괴]* (심각도: High) 
     수작업 진단 구조 상 접수가 늘어나면 운영 비용 기하 증가. 
     → **대응책**: 폼 입구(위젯) 자체를 B2B 타겟 오프라인 시범 매장(10곳) 코드로 철저히 막아 대국민 오픈(B2C 진단) 트래픽 인입을 개발 전까지 차단 조율.
  3. *[Risk 3: 부정 정보 노출에 충격받은 무작위 이탈]* (심각도: Mid) 
     사용자가 디자인만 보고 이탈 시 전환 저조 우려. 
     → **대응책**: 이탈 대상을 패션화에 매달리는 N1군으로 간주하여 이탈 자체를 '성공적 포지셔닝' 필터로 사용. (코어 C1만 전환하면 CPA 달성에 성공함).

## 7. 범위 체계 (In/Out 명확화 방침)
- **In Scope (1차 검증 집중 / PM 사수 영역)**:
  - 타겟을 글로벌 베스트셀러 운동화 Top 30 정보의 수동 DB 매핑(텍스트/이미지만 구성).
  - 페이크도어로 C1 CVR 트래픽 견인율 통제 실험 진행(No Code Carrd 결합).
- **Out of Scope (충돌 및 개발 연기 / 의사결정 배제 영역)**:
  - MVP 내 자체 인앱 결제/PG 연결 전면 불가 (모두 외부 연결로 분산).
  - 유저 리텐션 확보 명분의 리뷰 포럼, NRC 등 외부 데이터 마일리지 동기화 모듈 개발 일체 포함 제외.
